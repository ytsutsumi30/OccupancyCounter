## 実行指示

提案資料の体系化を行う。**実行（移動・リネーム・削除・上書き）は禁止、読み取りと提案のみ**。

対象フォルダ（4カ所すべてスキャン）:

1. `/mnt/c/Users/tsuts/OneDrive/ドキュメント/Grafana`
2. `/mnt/c/prj3/ai_agent`
3. `/mnt/c/PRJ2`
4. `/home/tsutsumi/grafana-setup-azure`

進め方:

- まず `find <各フォルダ> -type f \( -iname '*.pdf' -o -iname '*.pptx' -o -iname '*.docx' -o -iname '*.xlsx' -o -iname '*.md' \)` で資料ファイルだけを棚卸しし、一覧を見せてから読み込みに進むこと（コードファイルは読まない。2 と 3 はプロジェクトフォルダなので資料の発掘が目的）
- PDF は直接読む。.pptx / .docx は `python3 -m pip install python-pptx python-docx` の上でテキスト抽出して読む
- ファイル数が多い場合は更新日が新しい順に優先し、全読み込み前に件数と概算時間を報告すること

出力するもの:

1. **現状の資料マップ**: ファイル名／場所／テーマ／用途／最終更新／状態（現役・古い・重複疑い）の一覧表
2. **重複ペアの洗い出し**: 内容が重なる資料の組と、どちらを正とすべきかの判定理由
3. **欠落テーマの指摘**: 「本来あるべきなのにない」資料
4. **理想のフォルダ構成**: ツリー形式＋命名規則＋現状→理想の移動マッピング（4 フォルダに散らばったものをどこへ集約するかを含む）
5. **テンプレート集**: 使い回せるパーツ（構成雛形・定型文・図表）をそのままコピペで使える状態で抽出
6. **移行ステップ**: 優先度付き。破壊的操作（削除・統合）は必ず最後のフェーズに分離

ユーザーが「GO」を出すまで何も変更しないこと。

## 実行前の注意（ユーザー向け）

- **OneDrive ファイルオンデマンド**: `ドキュメント/Grafana` の実体がクラウドのみだと WSL から読めない。エクスプローラーでフォルダ右クリック →「このデバイス上に常に保持する」を推奨
- **大文字小文字**: `/mnt/c/prj3` と `/mnt/c/PRJ2` は実際のフォルダ名に合わせる（`ls /mnt/c/` で確認）
- 理想構成の叩き台（読了後に具体化される前提の汎用版）:

```
提案資料/
├── 00_テンプレート/          ← 完成版の雛形のみ
├── 10_業種別提案/            ← 顧客に出す完成品（<業種または顧客>/<YYYYMM_案件名>/）
├── 20_パーツ集/              ← 実績スライド・図版・データ部品
├── 30_過去案件アーカイブ/     ← YYYYMM_顧客名_結果(受注/失注)
└── 90_作業中/                ← 未完成品はここ以外に置かない
```

命名規則: 日付は先頭に `YYYYMM`。「最終」「fix」「(2)」は禁止（版はフォルダ内 1 本に集約、旧版は 30 へ）。
