# ループ設計テンプレート

ループを開始する前にこのテンプレートを埋める。埋められない項目がある場合はループ化しない（またはユーザーに確認する）。

## 0. 前提判断

- ループが必要な理由:（単発作業で済まないか？）
- ループの種類: ゴールベース / タイムベース / プロアクティブ

## 1. 目的（Goal）

- タスク:（例: FaceAnalyzer のユニットテストを直す）
- 機械判定可能なゴール:（例: `./gradlew test` が exit code 0 で完了する）
- 判定コマンド:（Verifier が毎回実行するコマンド。複数可）

## 2. トリガー

- 起動方式:（例: `/loops` の単発起動）

## 3. ループ本体

- Doer の作業単位:（1 イテレーションで加える最小の変更）
- Verifier の検証方法:（判定コマンドの実行。サブエージェント分離の要否）
- 未達時の方針:（差分修正 → 再実行。同じアプローチの繰り返し禁止）

## 4. 停止条件（必ず数値・明確な条件で）

- 成功条件:（例: 判定コマンドがすべて成功）
- 最大試行回数:（既定 5）
- エスカレーション条件:（例: 同一エラーで 2 回進捗なし / スコープ変更が必要）

## 5. 記録

- ログの場所:（scratchpad の `loop-log.md`）
- 記録内容: 試行番号 / 変更点 / 検証結果 / 次の方針

## 6. ガードレール

- 禁止事項:（例: テストの削除・skip 化、閾値の緩和、force push）
- 人間レビューが必要な操作:（例: 外部 API 仕様の変更、リリース）
